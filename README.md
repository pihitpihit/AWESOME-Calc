# AWESOME-Calc

PC 게임 *Satisfactory*를 위한 헬퍼 앱. 공장 설계 시 자주 마주치는 계산 — 레시피 트리 전개, 자원 수지, 처리량 밸런싱 등 — 을 보조한다.

---

## 비전

게임 플레이 중 "이 라인에 컨스트럭터 몇 대가 필요하지?", "이 빌딩을 짓는 데 자원이 얼마나 필요하지?" 같은 계산을 빠르게 답해주는 도구. iPad 위에서 직관적으로 다룰 수 있는 UI를 1차 목표로 한다.

## 구조

프로젝트는 두 부분으로 나뉜다.

```
+---------------------------+       +-----------------------------+
|  app (iPad)               |       |  cli                        |
|  React Native + TS        |       |  Python                     |
|  UI: pihitpihit/plastic   |       |  prototyping & test 자동화  |
+-------------+-------------+       +--------------+--------------+
              |                                    |
              |        공유 도메인 / 데이터        |
              +-----------------+------------------+
                                |
                       +--------v--------+
                       |  data/          |
                       |  recipes, items |
                       |  machines (JSON)|
                       +-----------------+
```

- **CLI**: 도메인 로직과 계산 알고리즘을 빠르게 프로토타이핑하고 회귀 테스트로 굳히는 자리.
- **앱**: CLI에서 검증된 로직을 iPad에서 실제로 쓸 수 있는 형태로 감싼 자리.
- **data/**: 두 파트가 같은 JSON을 바라보도록 단일 원천을 둔다.

도메인 모델은 CLI에서 먼저 정의·검증하고, 앱은 동일한 모델을 TypeScript로 옮긴다.

---

## CLI

- **언어**: Python 3.11+
- **플랫폼**: macOS / Linux / Windows (개발 기기 무관)
- **목적**
  - 계산 알고리즘 프로토타이핑 (레시피 그래프, 처리량 솔버 등)
  - 게임 데이터 정규화 파이프라인
  - 도메인 로직 회귀 테스트
- **주요 의존성 (예정)**
  - `typer` — CLI 프레임워크
  - `pydantic` — 도메인 모델 / 데이터 검증
  - `networkx` — 레시피 그래프
  - `pytest` — 테스트
- **패키징**: `uv` 또는 Poetry

### CLI가 책임지는 것
- 도메인 규칙 (레시피 적용, 단위 변환, 처리량 계산)
- 데이터 무결성 검증
- 시나리오 기반 회귀 테스트

### CLI가 책임지지 않는 것
- UI / UX
- 사용자 데이터 영속화 (앱이 담당)

---

## 앱

- **언어**: TypeScript
- **프레임워크**: React Native
- **UI**: [`pihitpihit/plastic`](https://github.com/pihitpihit/plastic) 컴포넌트 라이브러리를 디자인 시스템으로 사용
- **타겟**: iPad (1순위). iPhone은 부수적, Android는 비목표.
- **상태 관리**: TBD — Zustand 우선 검토
- **로컬 저장**: AsyncStorage 또는 SQLite (클라우드 동기화는 비목표)
- **빌드**: Expo / bare RN — 결정 보류

### 앱이 책임지는 것
- iPad에 최적화된 인터랙션 (큰 화면, 분할 뷰, 키보드 단축키)
- 사용자 시나리오 (공장 저장/불러오기, 결과 시각화)
- CLI에서 검증된 알고리즘의 TypeScript 포팅

## 도메인 / 데이터

- `data/` 디렉토리에 레시피·아이템·머신·자원 노드를 정규화된 JSON으로 둔다.
- CLI와 앱은 **같은 파일**을 소비한다.
- 1차 출처: Satisfactory 공식 데이터 덤프 또는 위키 (라이센스 확인 필요).

## 로드맵 (러프)

1. **Phase 0 — 기초**: 도메인 모델 초안, 게임 데이터 수집·정규화
2. **Phase 1 — CLI MVP**: 단일 아이템 처리량 계산기 + 테스트
3. **Phase 2 — CLI 확장**: 레시피 트리 솔버, 자원 수지
4. **Phase 3 — 앱 MVP**: plastic 컴포넌트로 단순 계산 화면 1개
5. **Phase 4 — 앱 확장**: 공장 저장, 결과 시각화, iPad 인터랙션 최적화
